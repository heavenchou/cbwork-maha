校勘條目新增標記規範

版本資訊：$Id: note_tag.htm,v 1.1.1.1 2003/05/05 04:04:55 ray Exp $

用詞說明：一筆校勘包含一至多組校勘，每一組校勘皆以全型逗點分隔。

 

影響整筆範圍的標記

<n> 本筆使用 <note> 標記
<x> 本筆使用 <note> 標記，並含在 <todo> 標記中
<a> 本筆不處理，使用 <app> 標記，並加上 xxxx 屬性記錄
<u> 本筆不處理，使用 <tt> 標記，並加上 xxxx 屬性記錄
<d> 本校勘會全部放入 desc 屬性中

影響一組範圍的標記

<?> 本組不處理，整筆記錄在 desc 屬性中，並使用 xxxx 註明本組沒有處理
<k> 本組是參考其它版本而自行加入的

<z> 手動加入的中文之前
<~> 沒有前導符號的梵巴文
<s> 沒有任何前導符號的梵文
<p> 沒有任何前導符號的巴利文

<m> 單組使用 note 註解
<e> 和某經相同的註解
<f> 參考某經的註解
<l> 註解 <lem> 的文字

<r> 以前是 resource ，現已作廢
其它

<o> 本標記之後的全部校勘皆放入 orig 屬性中
<c> 本標記之後的全部校勘皆放入 orig 屬性中
<t> 加在可疑文字之前

[A>B] 勘誤的處理
全型逗號 '，' 符號的處理 (當它出現在非分隔符號時)

 

影響整筆範圍的標記
標記 <n>
插入位置 整筆校勘條目最前面
說明 本校勘以後不再處理，只產生 note 格式的 xml 校勘，不再加 xxxx 屬性備忘。
範例 <n>知恩院本奧書
產生的ＸＭＬ格式 <note n="000101" place="foot">知恩院本奧書</note>
標記 <x>
插入位置 整筆校勘條目最前面
說明 本校勘被忽略還沒處理，並產生 note 格式的 xml 校勘，並使用 <todo> 標記。(以前寫：並以 xxxx 屬性備忘，這點已作廢)
範例 <x>此序依宋元明三本而載
產生的ＸＭＬ格式

<todo>

<note n="000101" place="foot">此序依宋元明三本而載</note>

</todo>
標記 <a>
插入位置 整筆校勘條目最前面
說明 本校勘被忽略不處理，並產生 app 格式的 xml 校勘，並以 xxxx 屬性備忘。
範例 <a>不分卷於此【明】
產生的ＸＭＬ格式

<app n="000101" desc="<a>不分卷於此【明】" xxxx="沒有處理的校勘">

<lem>???</lem>

<rdg wit="【???】">???</rdg>

</app>

標記 <u>
插入位置 整筆校勘條目最前面
說明 本校勘被忽略不處理，並產生 tt 格式的 xml 校勘，並以 xxxx 屬性備忘。
範例 <u>欲愛色愛無色愛∼Kaamata.nhaa（欲愛）,bhavata.nhaa（有愛）,vibhavata.nhaa（無有愛）
產生的ＸＭＬ格式

<tt n="000101" desc="<u>欲愛色愛無色愛∼Kaamata.nhaa（欲愛）,bhavata.nhaa（有愛）,vibhavata.nhaa（無有愛）" xxxx="沒有處理的校勘">

<term lang="zh">???</term>

<term lang="unknown">???</term>

<term lang="zh" place="foot">???</term>

</tt>
標記 <d>
插入位置 整筆校勘條目最前面
說明 本校勘皆要處理，但在 desc 屬性保留本筆校勘，讓使用者瞭解。
範例 <d>〔其衣…說〕十八字－【三】
產生的ＸＭＬ格式

<app n="000101" desc="〔其衣…說〕十八字－【三】" xxxx="此處自動處理…，請複查 ">

<lem>其衣…說</lem>(註:此處在插入 xml 經文時會自動尋找範圍)

<rdg wit="【三】">&lac;</rdg>

</app>

 

影響一組範圍的標記
標記 <?>
插入位置 某一組校勘條目的前面
說明 該組校勘被忽略不處理，在 desc 屬性保留本筆校勘，並以 xxxx 屬性備忘。
範例 熇＝稿【三】＊，<?>但熇稿交用【宋】
產生的ＸＭＬ格式

<app n="000101" desc="熇＝稿【三】＊，但熇稿交用【宋】" xxxx="有部份校勘沒處理 ">

<lem>熇</lem>

<rdg wit="【三】＊">稿</rdg>

</app>

標記 <k>
插入位置 某一組校勘條目的前面
說明 表示這一組是原來大正藏沒有的，是後來查出來的。若有相關的參考資料，則放在此標記後面。
範例

博＝[十*(十/田/寸)]【宋】，<k><resp="CBETA.Saywhat" cf1="K20p737a13">＝[(怡-台)*(十/田/寸)]【麗】
產生的ＸＭＬ格式

<app n="xxxxxx">

<lem>博</lem>

<rdg wit="【宋】">[十*(十/田/寸)]</rdg>

<rdg wit="【麗】" resp="CBETA.Saywhat" cf1="K20p737a13">[(怡-台)*(十/田/寸)]</rdg>

</app>

註：組字式在 xml 中實際上會轉成 &CB 碼
標記 <z>
插入位置 有些梵巴文字沒有中文，<z>就放在後來加入的中文之前。
說明 有些梵巴文字沒有中文，<z>就放在後來加入的中文之前。
範例

<z>瞿曇∼Gotama.
產生的ＸＭＬ格式 <tt n="000101" type="app">

<term lang="zh">瞿曇</term>

<term lang="pali">Gotama.</term>

</tt>

註：原本應採用勘誤模式來處理 [>瞿曇]，但因勘誤模式還沒有定案，故暫時如此表示
標記 <~>
插入位置 沒有∼符號的梵巴文字前 (適用於不確定是梵文或巴利文)
說明 某些梵巴文字沒有以∼符號為前導，無法由程式辨識，故加入此標記，以利處理。
範例

迦葉<~>Kassapa.
產生的ＸＭＬ格式 <tt n="000101" type="app">

<term lang="zh">迦葉</term>

<term lang="unknown">Kassapa.</term>

</tt>

註：此例是標準的格式，尚有其它格式可應用，其它請參考<s><p>
標記 <s> or <S>
插入位置 沒有任何符號的梵文前 (確定此字是梵文)
說明

某些梵文沒有任何符號為前導，無法由程式辨識，故加入此標記，以利處理。

大小寫的 <S> 標記在應用上並沒有差別，大寫主要是為了用在確定沒有相對應漢文的梵文中，另一支協助加入漢文的程式才不會針對 <S> 作用。
範例

增壹阿含<s>Ekottara-AAgama∼A^nguttara-Nikaaya.

產生的ＸＭＬ格式 <tt n="000101" type="app">

<term lang="zh">增壹阿含</term>

<term lang="sk">Ekottara-AAgama</term>

<term lang="pali">A^nguttara-Nikaaya.</term> (實際上有些字會改成 &xxx; 格式)

</tt>

註：此例是多組梵巴文的格式，尚有其它格式可應用，其它請參考<~><p>
標記 <p> or <P>
插入位置 沒有任何符號的巴利文前 (確定此字是巴利文)
說明

某些巴利文沒有任何符號為前導，無法由程式辨識，故加入此標記，以利處理。

大小寫的 <P> 標記在應用上並沒有差別，大寫主要是為了用在確定沒有相對應漢文的巴利文中，另一支協助加入漢文的程式才不會針對 <P> 作用。
範例

<p>Niga.n.tha-Naatha-putta.
產生的ＸＭＬ格式

<term n="000101" place="foot" lang="pali">Niga.n.tha-Naatha-putta.</term> (實際上有些字會改成 &xxx; 格式)

註：此例是沒有漢文的格式，尚有其它格式可應用，其它請參考<~><s>
標記 <m>
插入位置 某一組校勘條目的前面
說明 本組使用 note 註解，若本筆有 <app> 則緊接在 <app>...</app> 之後
範例

愛＝受【宋】【元】，<m>聖本有光明皇后願文
產生的ＸＭＬ格式

<app n="0001001a">

<lem>愛</lem>

<rdg wit="【宋】【元】">受</rdg>

</app>

<note n="0001001b" place="foot">聖本有光明皇后願文</note>

標記 <e>
插入位置 某一組校勘條目的前面
說明 本組使用 note 註解，標註其和某經為 "相同"，若本筆有 <app> 則緊接在 <app>...</app> 之後
範例

<e>∼S. 22. 12-14. Anicca, etc.
產生的ＸＭＬ格式 <note n="0001001" place="foot" type="equivalent">∼S. 22. 12-14. Anicca, etc.</note>
標記 <f>
插入位置 某一組校勘條目的前面
說明 本組使用 note 註解，表示參考後面的資料，若本筆有 <app> 則緊接在 <app>...</app> 之後
範例

<f>[No. 100(266)]
產生的ＸＭＬ格式 <note n="0001001" place="foot" type="cf.">[No. 100(266)]</note>
標記 <l>
插入位置 某一組校勘條目的前面
說明 本組使用 note 註解，但專門註解 <lem>後面的字
範例

愛＝受【宋】【元】，<l>明註曰愛南藏作受
產生的ＸＭＬ格式

<app n="0001001a">

<lem>

<note n="0001001b" place="foot">明註曰愛南藏作受</note>

愛

</lem>

<rdg wit="【宋】【元】">受</rdg>

</app>

 

其它
標記 <o>
插入位置 某一組的最前面，但影響範則由<o>直至本筆最末。
說明 因為部份校勘有修正，為了保留原始資料，故將原始資料放在<o>之後，xml 則會將<o>之後的資料全部放入 orig 的屬性之中。
範例

螫＝蝎【宋】，＝蠍【元】【明】，<o>螫＝蝎，【宋】＝蠍【元】【明】
產生的ＸＭＬ格式

<app n="xxxxxx" orig="螫＝蝎，【宋】＝蠍【元】【明】">

<lem>螫</lem>

<rdg wit="【宋】">蝎</rdg>

<rdg wit="【元】【明】">蠍</rdg>

</app>
標記 <c>
插入位置 某一組的最前面，但影響範則由<c>直至本筆最末。
說明 功能和<o>相同，但只用在左右交換的校勘條目，這是為了日後方便尋找此類條目。
範例

噎＝咽【明】， <c>咽＝噎【明】
產生的ＸＭＬ格式

<app n="xxxxxx" orig="咽＝噎【明】">

<lem>噎</lem>

<rdg wit="【明】">咽</rdg>

</app>
標記 <t>
插入位置 任何有疑問的字之前，不要在符號或標記之前。
說明 加入<t>表示此處有疑問，打算在日後需再行查證，xml 則改成 <todo/> 標記
範例

瞿<t>曇∼Gotama.
產生的ＸＭＬ格式 <tt n="000101" type="app">

<term lang="zh">瞿<todo/>曇</term>

<term lang="pali">Gotama.</term>

</tt>

勘誤 [Ａ>Ｂ]<resp="CBETA.xxx" cf="xxxx" .... >
位置 任何要勘誤的字。若有 resp 及相關的標記，則緊鄰在其後，resp 是必需的屬性。
說明 校勘之中勘誤，採用 corr 模式，並加上 place="foot"，以便和一般 corr 區別。
範例

貪欲延＝食福[輕>經]<resp="CBETA.Maha">【三】
產生的ＸＭＬ格式

<app n="xxxxxx">

<lem>貪欲延</lem>

<rdg wit="【三】">食福<corr sic="輕" place="foot" resp="CBETA.Maha">經</corr></rdg>

</app>

註：若是”貪欲延”這三字有錯，則採用 <o> 標記來處理

貧欲延＝食福經【三】，<o>貪欲延＝食福經【三】
全型逗號 非使用在分隔各組校勘的全型逗號處理方式
說明 全型逗號的作用為分隔各組校勘，若有其它地方也出弄逗號，則要額外處理。
實例

任何非分隔各組用的全型逗號，請將逗號（，）改成半型加標記的逗號（<,>），程式在處堙時會自動還原全型逗號。

＜原來的＞穰＝禳【宋】【元】，【明】【宮】【乙】＊
＜修正後＞穰＝禳【宋】【元】<,>【明】【宮】【乙】＊

<app n="xxxxxx">

<lem>穰</lem>

<rdg wit="【宋】【元】，【明】【宮】【乙】＊ ">禳</rdg>

</app>
