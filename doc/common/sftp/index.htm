SSH Secure File Transfer Client 介紹

by Heaven

1. 程式說明

SSH Secure File Transfer Client 是一個類似 FTP 的程式，但它和一般 FTP 不同，它有加密的功能，所以比較安全，不用擔心半路有人偷聽，所以也比較不會被竊取帳號及密碼。

當然，這是因為伺服器有支援這個功能，我們才會用這個程式上傳與下載，若對方沒支援，用這支程式也沒有用。

2. 安裝

我想安裝應該沒什麼特別困難，因為我沒有重新安裝，所以也無法將安裝畫面貼出來。

3. 第一次執行

首先要做書籤，選 Profiles -> Add Profile...

出現底下畫面後，輸入名稱，可用 intranet，並按下右邊的 Add to Profiles

然後再選 Profiles -> Edit Profile...

就會看到有一個 intranet 在左邊

然後在右邊的 Host name 填上 w3.cbeta.org

User name 填上 cbeta

如果是在 CBETA 辦公室，則 Host name 填上 10.1.6.14 應該會變的很快。

填完之後按 OK 即可。

這時再選 Profiles 就可以看到 intranet 的書籤，按下去即可執行。

第一次執行時，會有如下訊息，按 Yes 即可。

然後會要你輸入密碼，密碼我不想寫在這裡，

也就是 CB五個數字 那一個密碼。

然後按下 OK

密碼確認過之後，右邊就會看到伺服器的目錄，預設應該會看到 /home/cbeta

這裡先解釋一下，在 unix 的作業系統中，它不像 DOS/Windows 有磁碟機 C: D: E: 等名稱，

它都是從根目錄 / 開始，而且目錄分隔的斜線方向相反，其它就和 DOS/Windows 差不多了。

你們可以在左邊移動到 C:\cbwork\doc 的目錄，右邊移到 /var/web/intranet 的目錄，

可以直接在上方紅色圈起來的地方輸入，也可以像在檔案總管的操作方式一般，一層一層地進入。

到達目的後，可按下旁邊的 Add 按鈕，就會把這些位置記憶起來，下次就可直接在下拉選單中找到。

而最下方的視窗就是傳輸檔案時的進度畫面。

4. 傳送檔案

傳送檔案很簡單，就是在二個視窗拖曳檔案或目錄即可。

就如檔案總管或 Total Commander 的操作。

請各位小心，這裡也可以刪除檔案，刪除了就救不回來了。

5. 目錄說明

我們在網站上看到的 http://w3.cbeta.org/intranet/ 這個目錄，

就等於在 sftp 看到的這個目錄 /var/web/intranet/

這二個目錄的底下都是同步的。

例如我把 SSHSecureShellClient-3.2.0.exe 放在 /var/web/intranet/common/utility/SSHSecureShellClient-3.2.0.exe

這時在做網頁時，要設定的連結位置就是必須是

http://w3.cbeta.org/intranet/common/utility/SSHSecureShellClient-3.2.0.exe

這樣別人才能順利連結，這樣應該不難懂。
