CBETA XML P5 標籤集

by：淑儀　最近更新日期：2009/07/22

 

CBETA XML 文件目前使用 utf-8 Encoding，標記採用國際標準 TEI，在 TEI 裏找不到合適的才會自訂標記，或自定標記屬性。一個 CBETA XML 檔內部架構分為檔頭（Header）及本文（Text）兩部份：

一、XML 經文檔的整體架構介紹

<TEI>
　　<teiHeader>
　　　　<text>
　　　　　　<body> 正文
　　　　　　<back> 正文後資訊

（實例）

<?xml version="1.0" encoding="utf-8"?>
<TEI xmlns="http://www.tei-c.org/ns/1.0" xmlns:cb="http://www.cbeta.org/ns/1.0" xml:id="T01n0001">
<teiHeader>......</teiHeader>
　<text>
　　<body>......</body>
　　<back>......</back>
　</text>
</TEI>

（按此關閉實例內容）

二、依檔頭、本文兩大類介紹 CBETA 目前所使用的標記

1. 檔頭（Header）

　1.1 File Description 本電子檔書目資訊描述 （CBETT P5 實例）

<fileDesc> 檔案描述。包含對電子檔案的完整書目敘述。
　　 <titleStmt> 標題陳述。(題名陳述) 匯集關於作品標題與作品智慧內容責任歸屬的資訊。
　　　 <title> 標題
　　　 <author> 作者
　　　 <respStmt> 負責人陳述。當未使用作者、編輯者等特定元素，或元素內容不足時，在此補充說明內容所屬負責人的責任陳述。
　　　　 <resp> 包含一個詞彙，來描述個人智慧責任的類型。
　　　　 <name>
　　 <editionStmt> 版本陳述
　　　 <edition> 版本
　　　　 <date> 日期
　　　 <respStmt xml:id="resp1">　xml:id 是指提供一個獨特識別符碼，識別帶有該屬性的元素。
　　　　 <resp>
　　　　 <name> 　　　
　　 <extent> 卷數
　　 <publicationStmt> 出版描述
　　　 <distributor> 發行者／單位
　　　　 <name>
　　　　 <address>
　　　　　 <addrLine>
　　　　 <availability> 著作權狀態。提供關於文件可利用性的資訊，例如使用或發行上的任何限制、版權狀態等。
　　　　　 <p>
　　　　 <date> 日期
　　 <sourceDesc> 來源描述
　　　 <bibl> 書目資料
　　　　 <listWit>
　　　　　 <witness xml:id="wit1">【CBETA】</witness> 包含學術編輯註解中所指的單一版本描述，或由一個印記所參照的版本列表。
　　　　　 <witness xml:id="wit2">【卍續】</witness>

（按此關閉 file description 內容）

　1.2 Encoding Description 電子版與來源之間的關係描述 （CBETA P5 實例）

<encodingDesc> 來源描述
　　<projectDesc> 計畫描述
　　　<p xml:lang > 使用RFC3066的代碼，指出該元素內容的使用語言
　　 <charDecl > 文字宣告
　　　 <char xml:id= " " >
　　　　 <charName> 文字名稱
　　　　 <charProp> 文字性質
　　　　　 <localName> 當地定義
　　　　　 <value> 性質值　　　
　　　　 <mapping> 文字對應

（按此關閉 encoding description 內容）

　1.3 Profile Description 文件背景描述 （CBETA P5 實例）

<profileDesc> 文件背景描述
　 <langUsage> 語言使用
　　 <language> 個別敘述文本中使用的語言

（按此關閉 profile description 內容）

　1.4 Revision Description 修訂摘要記錄（CBETA P5 實例）

<revisionDesc > 修訂描述
　　 <change when > 變更內容與日期

（按此關閉 revision description 內容）
