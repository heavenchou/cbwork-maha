4 xml 轉 normal　▲ top

程式名稱：xml2txt.bat，connect.bat

程式位置：C:\cbwork\work\bin

程式用途：將xml檔轉成普及版。

程式步驟：

Step1. 產生普及版

C:\>xml8

C:\cbwork\xml\Xxx>xml2txt –v Xxx –u

Step2. 連結小檔

C:\>rel8

C:\release\normal\Xxx>connect

_ xml2txt 是會產生一個暫存檔(c:\cbwork\err.txt)，正常結束會刪除暫存檔。如果 xml2txt 結束了，暫存檔卻還在，就表示不正常結束。

_ 不同需求請加入不同的參數。

_ 跑56冊normal指令：(1) pannormal—有檔頭資訊 (2) hnormal—無檔頭資訊，再執行 connect。

4 簡單標記版(BM) 轉 normal　▲ top

程式名稱：pre_one.bat，nor_one.bat

程式位置：C:\cbwork\simple\bin

程式用途：將簡單標記版轉成普及版。

程式步驟：

Step1. 產生new4.txt

C:\cbwork\simple\bin>pre_one Xxx（不要經文資訊：pre_one_nohead Xxx）

Step2. 產生普及版

C:\cbwork\simple\bin>nor_one Xxx（不要經文資訊：nor_one_nohead Xxx）

Step3. 連結小檔

C:\>simco8

C:\cbwork\simple\release\normal\Xxx>connect

備註：

加 "nohead" , 結果是沒有 "通用字".
若移除 "nohead" , 結果就是有 "通用字".
如果又要組字式, 又要有卷首, 等於是要這樣執行
pre_one_h.bat T01 (或 pre_one.bat T01 nohead)
nor_one.bat T01

第一個要加 nohead , 其實只是不用通用字.
第二個不加 nohead , 就是呈現卷首資訊.

4 新舊XML批次檔案比對　▲ top

一、批次檔比對程式

FGFC.EXE：DOS 版檔案比對程式，適合批次執行。
FGFCSIGN.TXT：比對忽略字元參考檔，可將欲忽略字元打在這個文字檔案裡面。

FGFC-allxml.BAT：比對全部XML的批次檔

二、批次檔案比對流程：

1. 上 CVS update XML 最新資料夾。

2. 備份 XML，使用 zip-xml.pl

程式步驟：
_ 參數請修改 c:\cbwork\work\bin\config.pl
在 $xml_in_dir="c:/cbwork/xml";　　　　　 # xml 輸入目錄 的下面一列加上：
$xml_out_dir="e:/temp/xml"; # xml 輸出目錄
然後到 e 碟手動新增 temp 資料夾，
再到 DOS 模式下，
進到：c:\cbwork\work\bin
輸入指令：perl zip-xml.pl
輸出檔位置：c:\temp\xml\
備份位置：\\Cbeta-bdc\xml 重大修訂前備份\

3. 執行 Ray 程式。

4. 把原存在於 "c:/cbwork/xml"　裡的 xml 合併成一冊一檔。
透過批次檔：connectx_all.bat ←「比對檔案一」

5. 把輸出檔 "c:/release/new-xml" 裡的 xml 合併成一冊一檔。
透過批次檔：connectx_all_new.bat ←「比對檔案二」

6. 然後將FGFC及 批次檔 放在同一個資料夾，開始進行檔案比對 (透過批次檔：FGFC-allxml.BAT)。

7. 再用「特殊標記頻次統計」程式統計有發生差異的冊數及其差異類型。

8. 如果差異都屬正常，進行程式轉出的XML標記檢測。

9. 檢測無誤，再把 "c:/release/new-xml" 裡的檔案覆蓋到　"/cbwork/xml"　中。

10. 上 CVS commit 所有 XML 經文。

#  -x 悉曇字呈現方法: (預設使用轉寫)
#     -x 1 使用 entity &SD-xxxx;
#     -x 2 使用 ◇【◇】
#  -z 不使用通用字
Ｂ. [xml→normal] 使用參數：-u -v -x 2
    [sm→normal]  使用    ：pre_one_nohead、nor_one_nohead

二、批次檔案比對流程

執行步驟：(以下批次檔均點選2下直接執行)
